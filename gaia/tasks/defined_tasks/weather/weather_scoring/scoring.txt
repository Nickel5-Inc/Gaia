# Weather Forecasting Incentive Mechanism Specification

## Overview

This document specifies a complete, deterministic incentive mechanism for Aurora weather forecasting on Bittensor. The system employs a distributed data storage approach, minimizing validator storage requirements while creating an ensemble forecast superior to any individual prediction.

## 1. Core System Architecture

### 1.1 Validator Operations
- Validators retrieve GFS input data (0.25° resolution) from NOAA servers
- Validators convert GFS data to Aurora Batch format
- Validators query miners with identical input data
- Validators request specific variables/timesteps for scoring as ground truth becomes available
- Validators construct variable-specific ensemble forecasts from top-performing miners

### 1.2 Miner Operations
- Miners receive Aurora Batch input with current state (two timestamps)
- Miners run modified Aurora models to generate 10-day forecasts (40 timesteps)
- Miners store complete forecasts in their own databases
- Miners serve specific variables/timesteps upon validator request
- Time limit: 5 minute for serving specific variables

### 1.3 Data Flow
```
GFS Data → Validators → Miners → [Miners store forecasts locally]
                           ↓
                     Day 3 arrives
                           ↓
Validators → Request specific variables (T+72h) → Miners
                           ↓
Validators ← Score specific variables ← Miners
                           ↓
Validators → Request full forecasts from top miners → Miners
                           ↓
Validators ← Construct variable-specific ensembles ← Miners
```

```python
# Validators request specific variables at specific lead times
function RequestValidationData(miner_id, variables, lead_times):
    # Example request:
    # variables = ['surf_2t', 'surf_msl', 'atmos_z']
    # lead_times = [72] # hours from forecast start
    
    request = {
        'miner_id': miner_id,
        'variables': variables,
        'lead_times': lead_times
    }
    
    # Miners implement server endpoint:
    @endpoint('/get_variables')
    function ServeVariables(request):
        forecast = db.forecasts.find_latest(request.miner_id)
        response = {
            'forecast_id': forecast.id,
            'data': {}
        }
        
        # Extract only requested variables at requested lead times
        for var in request.variables:
            var_data = load_variable_data(forecast.variables[var])
            for lead_time in request.lead_times:
                timestep = lead_time // 6  # Convert hours to 6-hour steps
                response.data[var + '_' + str(lead_time)] = var_data[timestep]
        
        return response
```

## 4. Deterministic Scoring System

### 4.1 Variable-Specific Scoring
```python
function ScoreVariable(miner_prediction, ground_truth, variable, lead_time):
    """Calculate skill score for a specific variable at a specific lead time"""
    
    # Variable-specific normalization factors
    normalization = {
        'surf_2t': 1.5,    # Temperature (K)
        'surf_msl': 200.0, # Pressure (Pa)
        'atmos_z': 20.0,   # Geopotential (m)
        'surf_10u': 2.0,   # Wind (m/s)
        'surf_10v': 2.0,   # Wind (m/s)
        'atmos_t': 1.0,    # Temperature (K)
    }
    
    # Calculate RMSE
    rmse = CalculateRMSE(miner_prediction, ground_truth)
    
    # Normalize by variable-specific factor
    normalized_rmse = rmse / normalization.get(variable, 1.0)
    
    # Convert to skill score (1.0 = perfect, 0.0 = no skill)
    skill = max(0, 1.0 - normalized_rmse)
    
    # Apply lead time difficulty bonus
    lead_time_factor = 1.0 + (lead_time / 240) * 0.5  # Up to 50% bonus for day 10
    
    return skill * lead_time_factor
```

### 4.2 Aggregate Miner Scores
```python
function AggregateMinerScores(miner_id, validation_days=[3, 5, 10]):
    """Calculate overall score for a miner based on all validation days"""
    
    total_score = 0
    
    for day in validation_days:
        # Get validation configuration for this day
        validation_config = StrategicValidationSample(day)
        
        # Request data for validation
        miner_data = RequestValidationData(
            miner_id, 
            validation_config['primary_vars'], 
            validation_config['lead_times']
        )
        
        # Score each variable
        day_score = 0
        for var in validation_config['primary_vars']:
            var_score = ScoreVariable(
                miner_data[var], 
                ground_truth[var],
                var,
                validation_config['lead_times'][0]
            )
            
            day_score += var_score
        
        # Normalize by number of variables
        day_score /= len(validation_config['primary_vars'])
        
        # Add to total score with appropriate weight
        total_score += day_score * validation_config['weight']
    
    return total_score
```

## 5. Variable-Specific Ensemble Construction

### 5.1 Identify Top Miners Per Variable
```python
function GetTopMinersForVariable(variable, level=None, num_miners=5):
    """Identify best miners for a specific variable/level"""
    
    # Find variable-specific scores
    var_scores = {}
    for miner_id in active_miners:
        # Get specific scores for this variable (from previous validations)
        if level:
            score_key = f"{variable}_{level}"
        else:
            score_key = variable
            
        var_scores[miner_id] = miner_scores[miner_id].get(score_key, 0)
    
    # Sort miners by score for this variable
    sorted_miners = sorted(
        var_scores.items(),
        key=lambda x: x[1],
        reverse=True
    )
    
    # Return top N miners
    return [miner_id for miner_id, score in sorted_miners[:num_miners]]
```

### 5.2 Bayesian Model Averaging
```python
function ConstructVariableBMA(variable, top_miners, lead_times):
    """Construct Bayesian Model Average for a specific variable"""
    
    # Request full forecast data for this variable from top miners
    var_forecasts = {}
    var_weights = {}
    
    for miner_id in top_miners:
        # Request full forecast sequence for this variable
        forecast_data = RequestFullVariableData(miner_id, variable, lead_times)
        var_forecasts[miner_id] = forecast_data
        
        # Get miner's weight for this variable (based on historical skill)
        var_weights[miner_id] = CalculateVariableWeight(miner_id, variable)
    
    # Normalize weights
    total_weight = sum(var_weights.values())
    if total_weight > 0:
        normalized_weights = {
            miner_id: weight/total_weight for miner_id, weight in var_weights.items()
        }
    else:
        # Equal weights if no history
    normalized_weights = {
            miner_id: 1.0/len(top_miners) for miner_id in top_miners
        }
    
    # Construct weighted ensemble
    ensemble = {}
    for lead_time in lead_times:
        ensemble[lead_time] = 0
        for miner_id, forecast in var_forecasts.items():
            weight = normalized_weights.get(miner_id, 0)
            ensemble[lead_time] += forecast[lead_time] * weight
    
    return ensemble
```

### 5.3 Complete Ensemble Construction
```python
function BuildFullEnsembleForecast():
    """Construct complete ensemble from variable-specific BMA"""
    
    # Full forecast lead times (6-hour steps)
    lead_times = list(range(0, 241, 6))
    
    # Initialize ensemble forecast
    ensemble = {}
    
    # Key atmospheric variables with pressure levels
    for var, levels in [
        ('atmos_z', [500, 850]), 
        ('atmos_t', [850]), 
        ('atmos_u', [250, 850]), 
        ('atmos_v', [250, 850])
    ]:
        for level in levels:
            var_key = f"{var}_{level}"
            top_miners = GetTopMinersForVariable(var, level)
            ensemble[var_key] = ConstructVariableBMA(var_key, top_miners, lead_times)
    
    # Surface variables
    for var in ['surf_2t', 'surf_msl', 'surf_10u', 'surf_10v']:
        top_miners = GetTopMinersForVariable(var)
        ensemble[var] = ConstructVariableBMA(var, top_miners, lead_times)
    
    return ensemble
```

## 6. Storage-Efficient Implementation 

### 6.1 Validator Storage Requirements
```
1. Validators store:
   - Current forecast metadata: ~1 MB
   - Scoring data: ~10 MB per miner
   - Ensemble output: ~200 MB
   - Ground truth data: ~500 MB
   
2. Total per validator: ~50-100 GB maximum
   - Batch process 25 miners at a time
   - Release memory after scoring each batch
```

### 6.2 Network Transfer Optimization
```
1. Strategic validation request:
   - ~20-30 MB per miner ideally
   - Binary format over WebSockets
   
2. Full forecast request (only for top miners):
   - Request one variable at a time
   - Progressive loading for ensemble creation
   - Compression during transfer
```

## 7. Full Operational Workflow

### 7.1 Day 0: Forecast Generation and Subset Storage
```
1. Validators send identical GFS input to all miners
2. Miners run Aurora model to produce 10-day forecasts
3. Miners return complete forecasts to validators
4. Validators extract and store critical subsets (~60MB per miner)
5. Validators discard the full forecast after extracting subset
6. Validators store singular timestep for all miners
```

### 7.2 Day 3: First Validation Cycle
```
1. Validators obtain ground truth data for T+72h
3. Validators compare miner submissions with stored subsets
4. Miners who provide different data than originally stored are penalized
5. Validators score honest predictions against ground truth
```

### 7.3 Days 5 & 10: Continued Validation
```
1. Repeat validation process at days 5 & 10
2. Accumulate scores across all validation points
3. Update ensemble weights based on all available scores
4. Final miner scores based on weighted average of all validation days
```

### 7.4 Reward Distribution
```
1. Base participation reward:
   - 15% of total reward pool
   - Distributed to all responsive miners
   - Requires valid predictions within time limit

2. Performance-based reward:
   - 85% of total reward pool
   - Distributed proportionally to aggregated scores
   - Calculation:
     miner_reward = base_reward + (performance_pool * miner_score / sum(all_scores))
```
